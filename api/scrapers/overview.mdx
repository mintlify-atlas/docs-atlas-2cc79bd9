## Purpose

The scraper system is responsible for fetching version information from multiple app stores (Apple App Store, Google Play, Microsoft Store, and Huawei AppGallery). It provides a unified interface for extracting app metadata including:

- Current version number
- Last update date
- App icon URLs

## Architecture Components

The scraper system uses a Factory pattern combined with a service orchestrator to manage multiple store implementations.

## IScraper Interface

All scrapers implement the `IScraper` interface, ensuring consistent behavior across different store types.

```typescript src/lib/scrapers/types.ts
export interface IScraper {
  scrape(config: StoreConfig): Promise<ScrapeResult>;
}

export interface StoreConfig {
  id: string;
  type: 'google-play' | 'itunes' | 'microsoft' | 'huawei';
  packageId?: string;    // For Google Play
  appId?: string;        // For iTunes and Huawei
  productId?: string;    // For Microsoft Store
}

export interface ScrapeResult {
  version: string;
  lastUpdateDate: string;
  icon?: string;
  error?: string;
}
```

### Key Design Principles

1. **Single Responsibility**: Each scraper handles one store type
2. **Unified Interface**: All scrapers return the same data structure
3. **Error Isolation**: Failures in one scraper don't affect others
4. **Type Safety**: TypeScript ensures correct configuration per store

## ScraperFactory

The `ScraperFactory` class uses the factory pattern to instantiate the correct scraper based on store type.

```typescript src/lib/scrapers/factory.ts
export class ScraperFactory {
  static getScraper(type: StoreConfig['type']): IScraper {
    switch (type) {
      case 'itunes':
        return new AppStoreScraper();
      case 'google-play':
        return new GooglePlayScraper();
      case 'microsoft':
        return new MicrosoftStoreScraper();
      case 'huawei':
        return new HuaweiScraper();
      default:
        throw new Error(`Unsupported store type: ${type}`);
    }
  }
}
```

### Usage Example

```typescript
const scraper = ScraperFactory.getScraper('itunes');
const result = await scraper.scrape({
  id: 'ios-app',
  type: 'itunes',
  appId: '1234567890'
});
```

## ScraperService

The `ScraperService` orchestrates the entire scraping process, handling:

- Parallel scraping of multiple stores per app
- Retry logic with configurable attempts
- Icon fallback from any successful store
- Graceful error handling (returns 'N/A' instead of failing)
- Store URL generation

```typescript src/lib/scrapers/service.ts
export class ScraperService {
  async scrapeAll(apps: AppConfig[]): Promise<{ lastUpdate: string; apps: App[] }> {
    const results: App[] = [];

    for (const app of apps) {
      const appData: App = {
        id: app.id,
        name: app.name,
        orderId: app.orderId,
        icon: '',
        stores: [],
      };

      // Scrape all stores in parallel for this app
      const scrapePromises = app.stores.map(async (store) => {
        try {
          const scraper = ScraperFactory.getScraper(store.type);
          const result = await this.scrapeWithRetry(scraper, store);

          // Use icon from first successful scrape
          if (!appData.icon && result.icon) {
            appData.icon = result.icon;
          }

          return {
            id: store.id,
            name: store.name,
            version: result.version,
            lastUpdateDate: result.lastUpdateDate,
            url: this.getStoreUrl(store),
          };
        } catch {
          // Return N/A on failure instead of throwing
          return {
            id: store.id,
            name: store.name,
            version: 'N/A',
            lastUpdateDate: 'N/A',
            url: this.getStoreUrl(store),
          };
        }
      });

      appData.stores = await Promise.all(scrapePromises);
      results.push(appData);
    }

    return {
      lastUpdate: new Date().toISOString(),
      apps: results,
    };
  }
}
```

### Retry Mechanism

The service includes automatic retry logic (default 3 attempts) to handle transient network failures:

```typescript src/lib/scrapers/service.ts
private async scrapeWithRetry(
  scraper: IScraper,
  config: StoreConfig,
  retries = 3
): Promise<ScrapeResult> {
  let lastError;
  for (let i = 0; i < retries; i++) {
    try {
      return await scraper.scrape(config);
    } catch (error) {
      lastError = error;
    }
  }
  throw lastError;
}
```

## Execution Flow

1. **Configuration**: Apps are configured with store IDs (appId, packageId, productId)
2. **Factory Selection**: `ScraperFactory.getScraper()` creates the appropriate scraper
3. **Parallel Execution**: All stores for an app are scraped simultaneously
4. **Retry Logic**: Failed requests are retried up to 3 times
5. **Validation**: Response data is validated against Zod schemas
6. **Aggregation**: Results are combined into the final App structure
7. **Error Handling**: Failed scrapers return 'N/A' instead of breaking the entire process
