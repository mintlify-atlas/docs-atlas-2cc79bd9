## Overview

The `DashboardTable` component is the core component responsible for displaying app version information in a structured, responsive table format. It handles both desktop (table view) and mobile (card view) layouts, and includes loading states with skeleton placeholders.

**Location**: `src/components/DashboardTable.tsx`

## Purpose

The DashboardTable component:

- Displays app version data for multiple apps and stores
- Provides a responsive design (table for desktop, cards for mobile)
- Shows app icons, names, store links, versions, and last update dates
- Includes loading states with skeleton UI
- Integrates with VersionBadge for version display

## Props

<ParamField path="apps" type="App[]" required>
  Array of app objects containing app and store information. Each app includes:
  - `id`: Unique identifier
  - `name`: App name
  - `orderId`: Display order
  - `icon`: App icon URL
  - `stores`: Array of store objects with version data
</ParamField>

<ParamField path="isLoading" type="boolean" required>
  Loading state indicator. When `true`, displays skeleton placeholders instead of actual data.
</ParamField>

### Row Spanning

The component uses `rowSpan` to merge cells for apps with multiple stores:

```tsx src/components/DashboardTable.tsx
{index === 0 && (
  <TableCell rowSpan={app.stores.length} className="px-3 py-2.5 align-top">
    <div className="flex items-start gap-2">
      {/* App icon and name */}
    </div>
  </TableCell>
)}
```

This creates a visual grouping where the app name/icon spans multiple rows (one per store).

### Icon Handling

The component includes fallback handling for app icons:

```tsx src/components/DashboardTable.tsx
{app.icon ? (
  <Image
    src={app.icon}
    alt={app.name}
    width={24}
    height={24}
    className="app-icon"
    unoptimized
    onError={(e) => (e.currentTarget.style.display = 'none')}

  <div className="app-icon bg-muted text-muted-foreground flex items-center justify-center text-xs font-bold">

```

Versions are displayed using the `VersionBadge` component with special handling for missing versions:

## Usage Examples

### Basic Usage

```tsx page.tsx
import { DashboardTable } from '@/components/DashboardTable';
import { App } from '@/lib/types';

function Dashboard() {
  const [apps, setApps] = useState<App[]>([]);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    fetchApps().then((data) => {
      setApps(data.apps);
      setIsLoading(false);
    });
  }, []);

  return <DashboardTable apps={apps} isLoading={isLoading} />;
}
```

### With Data Fetching

```tsx
'use client';

import { useState, useEffect } from 'react';
import { DashboardTable } from '@/components/DashboardTable';
import { DashboardData } from '@/lib/types';

export default function DashboardPage() {
  const [data, setData] = useState<DashboardData | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  const fetchDashboardData = async () => {
    setIsLoading(true);
    try {
      const response = await fetch('/api/dashboard');
      const dashboardData = await response.json();
      setData(dashboardData);
    } finally {
      setIsLoading(false);
    }
  };

  useEffect(() => {
    fetchDashboardData();
  }, []);

  return (
    <DashboardTable
      apps={data?.apps || []}
      isLoading={isLoading}
    />
  );
}
```

## Loading State

The component includes a comprehensive `TableSkeleton` component that displays placeholder content during loading:

```tsx src/components/DashboardTable.tsx
function TableSkeleton() {
  return (
    <div className="space-y-4">
      {/* Mobile skeleton */}
      <div className="space-y-4 md:hidden">
        {Array.from({ length: 4 }).map((_, i) => (
          <Card key={i} className="border-muted overflow-hidden shadow-sm">
            <div className="bg-muted/30 flex items-center gap-3 px-4 py-3">
              <Skeleton className="h-8 w-8 rounded-md" />
              <Skeleton className="h-4 w-32" />
            </div>
            {/* More skeleton content */}
          </Card>
        ))}
      </div>

      {/* Desktop skeleton */}
      <div className="hidden md:block">
        {/* Skeleton table rows */}
      </div>
    </div>
  );
}
```

The skeleton automatically shows:
- 4 card placeholders on mobile
- 8 table row placeholders on desktop
- Skeleton components for icons, text, and badges
