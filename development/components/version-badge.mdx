## Overview

The `VersionBadge` component is a specialized badge that displays app version numbers with click-to-copy functionality. It provides visual feedback when a version is copied to the clipboard, enhancing the user experience when working with version information.

**Location**: `src/components/VersionBadge.tsx`

## Purpose

The VersionBadge component:

- Displays version numbers in a styled badge format
- Provides one-click copying of version strings to clipboard
- Shows visual confirmation when version is copied
- Uses monospace font for better version number readability
- Integrates seamlessly with the DashboardTable component

## Props

<ParamField path="version" type="string" required>

## Implementation

### Component Structure

The component is a client-side component (marked with `'use client'`) that manages its own state for the copy feedback:

```tsx src/components/VersionBadge.tsx
'use client';

import { useState } from 'react';
import { Badge } from '@/components/ui/badge';
import { cn } from '@/lib/utils';
import { Check } from 'lucide-react';

export function VersionBadge({ version, className }: VersionBadgeProps) {
  const [copied, setCopied] = useState(false);

  const handleCopy = async () => {
    try {
      await navigator.clipboard.writeText(version);
      setCopied(true);
      setTimeout(() => setCopied(false), 1000);
    } catch (err) {
      console.error('Failed to copy version:', err);
    }
  };

  return (
    <Badge
      variant="secondary"
      className={cn(
        'hover:bg-muted-foreground/20 cursor-pointer transition-all duration-200 active:scale-95',
        copied && 'bg-green-100 text-green-700 dark:bg-green-900/30 dark:text-green-400',
        className
      )}
      onClick={handleCopy}
      title="Click to copy version"
    >
      <span className={cn('flex items-center gap-1.5', copied && 'font-semibold')}>
        {copied ? (
          <>
            <Check className="h-3 w-3" />
            Copied!
          </>
        ) : (
          version
        )}
      </span>
    </Badge>
  );
}
```

### Copy Functionality

The component uses the Clipboard API to copy version text:

```tsx
const handleCopy = async () => {
  try {
    await navigator.clipboard.writeText(version);
    setCopied(true);
    setTimeout(() => setCopied(false), 1000);
  } catch (err) {
    console.error('Failed to copy version:', err);
  }
};
```

**Key features:**
- Uses `navigator.clipboard.writeText()` for copying
- Sets `copied` state to `true` immediately after copy
- Resets `copied` state after 1 second
- Includes error handling for clipboard access failures

### Visual States

The component has two visual states:

1. **Normal State**: Displays the version number
   - Uses secondary badge variant
   - Shows hover effect
   - Includes "Click to copy version" tooltip

2. **Copied State**: Shows confirmation
   - Changes background to green
   - Displays checkmark icon
   - Shows "Copied!" text
   - Makes text bold

**Style breakdown:**

When copied, the badge applies green styling:

- **Light mode**: `bg-green-100 text-green-700`
- **Dark mode**: `dark:bg-green-900/30 dark:text-green-400`
- **Font weight**: `font-semibold` for the text

## Browser Compatibility

The Clipboard API is supported in all modern browsers:

- Chrome/Edge 63+
- Firefox 53+
- Safari 13.1+

**Note**: The Clipboard API requires a secure context (HTTPS or localhost).

### Fallback Considerations

For older browsers, you might want to implement a fallback:

```tsx
const handleCopy = async () => {
  try {
    if (navigator.clipboard) {
      await navigator.clipboard.writeText(version);
    } else {
      // Fallback for older browsers
      const textArea = document.createElement('textarea');
      textArea.value = version;
      document.body.appendChild(textArea);
      textArea.select();
      document.execCommand('copy');
      document.body.removeChild(textArea);
    }
    setCopied(true);
    setTimeout(() => setCopied(false), 1000);
  } catch (err) {
    console.error('Failed to copy version:', err);
  }
};
```

## Testing Considerations

When testing this component:

1. **Mock Clipboard API**: The Clipboard API needs to be mocked in tests
   ```tsx
   Object.assign(navigator, {
     clipboard: {
       writeText: jest.fn(() => Promise.resolve()),
     },
   });
   ```

2. **Test Copied State**: Verify the copied state appears and disappears
   ```tsx
   await userEvent.click(badge);
   expect(screen.getByText('Copied!')).toBeInTheDocument();
   await waitFor(() => {
     expect(screen.queryByText('Copied!')).not.toBeInTheDocument();
   }, { timeout: 1100 });
   ```

3. **Test Error Handling**: Simulate clipboard access failure
   ```tsx
   navigator.clipboard.writeText = jest.fn(() => Promise.reject());
   ```
